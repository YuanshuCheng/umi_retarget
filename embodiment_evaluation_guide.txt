================================================================================
FastUMI Retarget Tool — 本体设计评估指南
================================================================================

目标: 评估一个具身机器人的本体设计能否完成特定的操作任务。

核心思路:
  人类演示动作 → retarget 到目标机器人 → 真机 replay → 分析执行情况
  如果 retarget 后机器人无法完美复现动作, 说明本体设计存在局限。


================================================================================
一、评估流程
================================================================================

Step 1: 选取典型动作采集
  │
  ├── 确定要评估的任务 (如: 抓瓶子放桌上、开抽屉、擦桌子)
  ├── 每个任务采集 5-10 条 FastUMI 数据
  ├── 覆盖不同难度:
  │     标准速度 (standard): 慢速、精确
  │     快速 (fast): 接近实际使用速度
  │     不同位置: 正前方、侧方、低位、高位
  └── 采集时注意:
        操作者的动作范围不要刻意受限
        自然完成任务, 不要为了机器人而降低动作幅度

Step 2: Kinematic Retarget
  │
  ├── fastumi-retarget init → 配置 URDF + 关节映射
  ├── fastumi-retarget batch → 批量优化
  ├── fastumi-retarget evaluate → 质量评估
  └── 输出: retarget 后的关节轨迹 + 质量报告

Step 3: Sim 验收 (仿真检查)
  │
  ├── 在 MuJoCo/viser 中播放 retarget 结果
  ├── 主观观察:
  │     动作是否自然、流畅
  │     有没有异常姿态 (过度弯腰、手臂扭曲、碰撞)
  │     关键动作 (抓取/释放) 是否到位
  └── 如果 sim 中就不合理 → 本体设计或 retarget 参数需要调整

Step 4: 真机 Replay
  │
  ├── 将 retarget 数据发送到真机执行
  ├── 记录 replay log (q_cmd vs q_actual)
  ├── 同时主观观察:
  │     机器人动作的稳定程度
  │     有没有停顿、抖动、猛追
  │     重心是否稳定 (轮式底盘是否倾斜)
  │     夹爪是否在正确位置开合
  └── 安全: 首次 replay 降速 (0.5x), 确认安全后原速

Step 5: 数据分析
  │
  ├── analyze_dynamics_error.py 分析 replay log:
  │     轨迹重合度 (actual vs cmd)
  │     夹爪-位姿匹配度
  │     各关节跟踪误差
  │     底盘/躯干稳定性
  └── 输出: 定量的执行误差报告


================================================================================
二、评估指标
================================================================================

A. Retarget 质量指标 (Step 2, 纯计算)
--------------------------------------------------------------------
  这些指标衡量: 优化器能否在当前本体约束下找到好的解

  1. EE 轨迹重合度 (cm)
     retarget 后的 EE 轨迹 vs 人类演示的 EE 目标轨迹
     < 2cm: 优秀, 本体可达性好
     2-5cm: 一般, 某些位置受限
     > 5cm: 差, 本体无法覆盖人类的运动范围

  2. 关键帧 EE 精度 (cm)
     抓取/释放时刻的 EE 位置误差
     < 1cm: 满足操作精度
     > 3cm: 操作可能失败

  3. 自碰撞帧比例 (%)
     优化后是否有自碰撞
     > 5%: 本体设计在该任务的运动空间有冲突

  4. 关节限位利用率 (%)
     关节角度占限位范围的百分比
     > 90%: 接近极限, 裕度不足

  5. 姿态自然度评分
     躯干前倾范围、肘部弯曲、腕关节使用率
     过度前倾 > 60°: 重心不稳风险
     肘部接近奇异: 力臂不足


B. 真机执行指标 (Step 5, 需要真机)
--------------------------------------------------------------------
  这些指标衡量: 真机能否精确执行 retarget 的轨迹

  1. 轨迹重合度 (cm)
     actual EE 轨迹 vs cmd EE 轨迹
     < 2cm: 跟踪良好
     > 5cm: 电机跟不上或控制器有问题

  2. 夹爪-位姿匹配度 (%)
     在空间最近位置下, 夹爪开合状态是否一致
     > 95%: 夹爪时序正确
     < 80%: 夹爪和手臂有相位差

  3. 各关节跟踪误差 (度)
     哪些关节跟踪差 → 定位硬件瓶颈
     特别关注躯干关节 (惯性大、响应慢)

  4. 底盘/重心稳定性
     底盘位置估计漂移
     躯干前倾导致的重心偏移


C. 主观观察项 (Step 3+4, 人工判断)
--------------------------------------------------------------------
  定量指标无法覆盖所有问题, 需要人工观察:

  □ 动作整体是否流畅自然
  □ 有没有不符合预期的姿态 (如: 本该侧身却全靠弯腰)
  □ 关键操作动作是否准确 (抓取位置、力度感)
  □ 机器人是否有倾倒风险
  □ 速度是否满足任务需求
  □ 有没有异响或异常振动

================================================================================
四、常见本体设计局限及表现
================================================================================

局限                    Retarget 表现                真机表现
----------------------------------------------------------------------
手臂太短               EE 重合度差 (>5cm)           够不到目标位置
                       优化器强制弯腰补偿

躯干自由度不够          过度前倾或侧倾               重心不稳
                       姿态不自然                    有倾倒风险

缺少侧摆自由度          侧向任务 EE 误差大           底盘大幅横移补偿
                       需要底盘大幅度移动

关节限位太窄            关节利用率 >90%              接近限位时速度变慢
                       优化器找不到好的解            操作空间受限

没有夹爪               无法检测关键帧                抓取任务无法完成
                       无夹爪匹配度指标

躯干关节响应慢          (retarget 看不出)             停顿-猛追
                                                    feedback 冻结

底盘精度差              (retarget 看不出)             轨迹漂移
                                                    里程计累积误差


================================================================================
五、使用建议
================================================================================

1. 先在 sim 中确认 retarget 质量, 再上真机
   避免: retarget 就不行的数据浪费真机时间

2. 典型任务选取要有代表性
   包含: 远距离够取、低位操作、侧向操作、快速动作
   不要只测简单的正前方操作

3. 每台新机器人建议先测 3 个标准任务
   正前方抓取 + 侧向抓取 + 低位操作
   快速评估手臂可达性 + 躯干灵活度 + 重心稳定性

4. retarget 参数需要为不同机器人调整
   R1Pro 的默认权重不一定适合 G2
   特别是躯干相关的 cost (tilt/yaw/lean)

5. 对比不同本体设计
   同一批采集数据, retarget 到不同 URDF
   直接比较 EE 重合度和姿态自然度
   → 定量说明哪个本体设计更适合该任务
